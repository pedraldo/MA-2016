Création d'un compte GitHub
Comment se créer un compte ?

Avant de commencer à utiliser Git ou même GitHub, il faut commencer par se créer un compte. GitHub propose 4 différentes solutions qui varient par la protection et la confidentialité des projets :

  • Offre gratuite : les projets sont visibles et téléchargeables par tous les utilisateurs.
  • Offre personnelle : possibilité de créer des projets privés visibles seulement par les membres invités.
  • Offre organisation : possibilité de gérer les utilisateurs avec des niveaux de permissions.
  • Offre entreprise : possibilité d’héberger les projets sur ses propres serveurs avec des protocoles de sécurité propre à l’entreprise.

Vu que nous débutons, nous allons commencer avec un compte simple. Rendez vous donc sur la page d'accueil :


Il ne vous reste plus qu’à vous trouver un identifiant (1), renseigner votre adresse mail (2), vous trouver un mot de passe (3) et on valide (4) !

Vous accèderez à une page vous permettant de choisir le type de votre compte (les offres présentées précedemment). Ici nous n’avons rien à faire, il suffit de valider et c’est terminé. Vous avez enfin votre compte GitHub. Vous allez pouvoir créer vos propres projets ou alors rejoindre les projets des membres de la communauté.

Vous allez ainsi installer deux programmes :

  • Git GUI : une interface graphique pour Git offrant toutes les fonctionnalités de la version console ainsi que des fonctionnalités visuelles

  • Installer à partir d’un paquet existant sur votre ordinateur
Nous vous conseillons d’installer Git depuis les sources, en vous dirigeant sur la page suivante.

Configuration Git

Vous avez la possiblité de spécifier les paramètres de configuration de Git grâce à la commande git config. Nous vous conseillons encore une fois de configurer Git après avoir créer votre compte Github, dans le but d’avoir certaines configurations similaires.

Grâce à cette commande et aux divers paramètres qu’elle propose, il possible de configurer beaucoup de paramètres dans le but de personnaliser votre utilisation de Git, principalement au niveau client. Dans notre cas, nous allons vous présenter quelques configurations qui sont basiques et pratiques pour une première utilisation de Git.

Génération des clés publiques SSH

Les clés SSH sont un moyen d'identifier les ordinateurs de confiance sans impliquer les mots de passe. Les serveurs Git utilisent ce genre d’authentification. Vous pouvez générer une clé SSH et ajouter la clé publique à votre compte GitHub en suivant les procédures décrites dans cette section.
Si vous souhaitez en savoir plus si le concept et l’utilisation des clés SSH, voici un lien utile.

Le processus de génération de clé SSH est identique sur tous les systèmes d’exploitation.

Tout d’abord, vérifiez que vous n’avez pas déjà créer de clé SSH auparavant.
Pour cela ouvrez un terminal (GitBash pour Windows) et taper la commande suivante :
ls -al ~/.ssh

Vérifiez si ce répertoire contient les fichiers suivants : id_dsa.pub ou id_rsa.pub.
